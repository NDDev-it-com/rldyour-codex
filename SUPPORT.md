# Support

Use public GitHub Issues for this adapter for:

- plugin, workflow, or runtime behavior defects,
- validation failures,
- documentation or packaging drift.

For security reports, use a private advisory at the repository security page.
Do **not** open public issue text for exploit details.

When filing an issue, include:

- environment details,
- command outputs from failing validation scripts,
- commit SHA,
- sanitized logs and minimal repro steps.
