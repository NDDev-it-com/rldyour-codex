# AGENTS.md - rldyour-codex

This repository is the Codex adapter package for the rldyour AI CLI control plane.
It owns Codex plugin manifests, managed subagents, MCP transport descriptors,
install/rollback scripts, and static validators. It does not vendor Codex runtime
binaries.

## Source of Truth

- `references/codex-baseline.json`
- `config/mcp-runtime-versions.env`
- `config/rldyour-contract.json`
- `plugins/**`
- `.agents/plugins/marketplace.json`
- `.codex-plugin/marketplace.json`
- `.codex-plugin/` plugin manifests
- `scripts/*.py`, `scripts/*.sh`
- `README.md`, `CHANGELOG.md`, `VERSION`

## Native Boundaries

Use Codex-native surfaces only:

- `$CODEX_HOME/*.toml` generated by installer scripts
- `.codex-plugin/` manifests
- `.agents/` managed agent definitions
- `.rules/` security/behavior rule files
- `plugins/**` metadata and runtime hooks
- GitHub workflow and validator surfaces in `.github` and `scripts`

Do not copy Codex-managed syntax into OpenCode/Gemini surfaces.

## Validation Commands

After any change in this module:

```bash
bash scripts/validate_fast.sh
python3 scripts/validate_release.sh
python3 scripts/validate_runtime.sh --strict-runtime
python3 scripts/validate_instruction_docs.py
python3 scripts/validate_plugin_versions.py
python3 scripts/validate_contract.py
```

At minimum, also run:

```bash
bash scripts/validate_fast.sh
python3 scripts/validate_execpolicy_rules.py
```

## MCP and Runtime Policy

- MCP launch metadata source is `.mcp.json` (portable metadata) and
  `config/mcp-runtime-versions.env` (runtime pins).
- Runtimes and launchers are validated through local smoke checks before commit.
- `install_system_codex.sh` writes portable paths into `~/.codex/config.toml`.

## Release and Control

1. Commit adapter changes in module.
2. Push `main` and release tags when `VERSION`/public contract changes.
3. Update super-repo tuple only after release and contract freshness checks pass.
4. Keep `.serena/` durable state current; runtime-local artifacts are gitignored.

## Security Policy

- Keep command surfaces and secrets model minimal and explicit in installer scripts.
- Never commit credentials, browser cookies/profiles, or temporary diagnostic traces.
- Security, CodeQL, secrets scanning, and dependency checks are mandatory in CI.
