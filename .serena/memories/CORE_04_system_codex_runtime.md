# CORE_04_system_codex_runtime

## Purpose

This memory records the installed system Codex runtime state for this repository, with emphasis on source-of-truth files, installed config shape, and verification commands.

## Source Of Truth

- `.agents/plugins/marketplace.json`: active marketplace catalog.
- `plugins/*/.codex-plugin/plugin.json`: plugin metadata and boundaries.
- `plugins/rldyour-mcps/.mcp.json`: MCP transport source-of-truth.
- `system/AGENTS.md`: tracked canonical global instruction template.
- `scripts/install_system_codex.sh`: installer for config, marketplace, plugins, MCP, and cache.
- `scripts/doctor_system_codex.sh`: installed-state verification.
- `scripts/validate_marketplace.sh`: repository/runtime consistency check.
- `scripts/smoke_mcp_runtime.sh`: installed MCP runtime smoke.
- `scripts/smoke_mcp_capabilities.*`: MCP tool-level smoke through `mcp` SDK.
- `scripts/bootstrap_check.sh`, `scripts/smoke_clean_bootstrap.sh`: bootstrap/install smoke.
- `scripts/smoke_hooks.sh`: hook smoke and temporary lifecycle verification.
- `scripts/check_mcp_runtime_versions.py`: pinned package freshness policy.
- `${CODEX_HOME:-$HOME/.codex}/config.toml`: active machine config.
- `${CODEX_HOME:-$HOME/.codex}/plugins/cache/rldyour-codex`: plugin cache used by Codex.
- `.github/workflows/validate.yml`: portable CI validation matrix.
- `${CODEX_HOME:-$HOME/.codex}/AGENTS.md`: installed global instruction output.

- `scripts/install_system_codex.sh --dry-run` (verify before write).

### Marketplace and cache behavior

- `marketplaces.rldyour-codex` is configured as local source and points to repo root.
- Plugin sources remain in repository files; runtime cache is populated under `plugins/cache/rldyour-codex/<plugin>/local`.
- Installed `~/.codex/AGENTS.md` is expected to equal `system/AGENTS.md` and describes fullrepo/rollback/verification behavior.
- Installed config is regenerated by reading:
  - `.agents/plugins/marketplace.json`
  - plugin manifests
  - `plugins/rldyour-mcps/.mcp.json`
  - environment overrides `UVX_BIN`, `BUNX_BIN`, `DART_BIN`, and `CODEX_HOME`.

### Runtime values and portability

- The installed config resolves portable command names in `.mcp.json` to machine-specific executables for:
  - `uvx` (`UVX_BIN` or `PATH`) for `serena`, `semgrep`.
  - `bunx` (`BUNX_BIN` or `PATH`) for `sequential-thinking`, `playwright`, `chrome-devtools`, `context7`, `shadcn`.
  - `dart` (`DART_BIN` or `PATH`) for `dart-flutter`.
- `scripts/validate_marketplace.sh` allows this path-resolution difference but requires identical names/args/URLs/timeouts/env settings between `.mcp.json` and installed config.

## Invariants

- Do not commit `${CODEX_HOME:-$HOME/.codex}/config.toml` or plugin cache as repository content.
- Do not store raw credentials or tokens in repo files or memories.
- Installed state is derived from repository sources; do not hand-edit config as the source of truth.
- Use `scripts/install_system_codex.sh --apply` after any change to manifests, `system/AGENTS.md`, `.mcp.json`, marketplace list, or installer config.
- Restart Codex when system state changes.
- Keep `system/AGENTS.md` and `AGENTS.md` aligned as template/source relationship.
- Keep diagnostics under ignored `diagnostics/`.

## Change Rules

- Change plugin catalog in `.agents/plugins/marketplace.json` and matching manifests via `plugin-creator`.
- Keep portable `.mcp.json` commands (`uvx`, `bunx`, `dart`) and pin versions.
- Use `scripts/bootstrap_check.sh --apply` for full machine setup smoke.
- Use `scripts/smoke_clean_bootstrap.sh` only on clean working tree to validate committed source-of-truth state.
- Prefer `scripts/doctor_system_codex.sh` after every installer change.
