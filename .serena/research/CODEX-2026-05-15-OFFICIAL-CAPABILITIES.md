# Codex 2026-05-15 Official Capabilities

## Purpose

This archive records the official Codex facts used to adapt the Claude-side
memory-brain workflow into Codex-native surfaces in `rldyour-codex`.

## Sources

- https://developers.openai.com/codex/hooks
- https://developers.openai.com/codex/plugins/build
- https://developers.openai.com/codex/config-reference
- https://developers.openai.com/codex/skills

## Verified Facts

- Codex hooks are enabled through `[features].hooks`; `features.codex_hooks` is
  only a deprecated alias and should not be generated by this repository.
- Codex lifecycle events used here are `SessionStart`, `PreToolUse`,
  `PermissionRequest`, `PostToolUse`, `UserPromptSubmit`, and `Stop`.
- `PreToolUse`, `PostToolUse`, and `PermissionRequest` match tool names.
  `UserPromptSubmit` and `Stop` ignore matcher. `SessionStart` matchers are
  startup/resume/clear oriented.
- Multiple hook entries can match the same event. Repository smoke tests must
  select expected hook entries by script path instead of assuming one hook per
  event/matcher.
- Codex plugin manifests live at `.codex-plugin/plugin.json`. Bundled hooks can
  be declared from plugin manifests and can point to `hooks.json`.
- Codex skills live under `skills/`. This repository uses `SKILL.md` for skill
  instructions and `skills/*/agents/openai.yaml` for Codex UI/dependency
  metadata.
- Claude Code plugin-root `agents/*.md` files and Claude-only `SKILL.md`
  frontmatter fields such as `allowed-tools` are not Codex-native surfaces in
  this repository.
- Codex has a built-in memories feature in config, but this repository keeps
  project knowledge in Serena `.serena/memories/` because Serena memories are
  already the cross-agent project knowledge source and can be published through
  `fullrepo`.

## Implementation Impact

- Added `scripts/validate_agent_tools.py` to reject Claude-only agent/skill
  surfaces and validate managed Codex TOML agents plus `agents/openai.yaml`
  MCP dependencies.
- Added `plugins/rldyour-serena-mcp/scripts/analyze_sync_scope.py` with Codex
  area names such as `CODEX-*`, numbered memory targets, and analyzer schema
  version 2.
- Updated `serena-memory-sync` to use `AREA-01-SLUG.md` memory names and
  `CORE-01-INDEX.md` as the active map.
- Updated `stop_memory_sync.sh` to route stale memory work to managed Codex
  `serena-sync` when delegation is available and to avoid Claude Code
  `Agent(...)` syntax.
- Updated `scripts/smoke_hooks.sh` for multiple matching hooks and added
  `scripts/smoke_serena_memory_taxonomy.sh` for memory-brain regressions.

## Reuse Rule

For future Codex behavior changes, re-check official OpenAI Codex docs first
through `openaiDeveloperDocs` or official OpenAI pages, then update validators,
skills, hooks, and Serena memories from verified facts.
